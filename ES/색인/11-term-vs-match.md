# term vs match

---

## 간단정리

---

- `match` : 검색어를 analyzer 로 쪼갠 뒤, 역색인과 비교한다. -> `text` 필드용
- `term` : 검색어를 쪼개지않고 그대로 역색인과 비교한다. -> `keyword` 필드용

|        | term      | match    |
|--------|-----------|----------|
| 검색어 분석 | ❌ 안함 (통짜) | ✅ 함 (쪼갬) |
| 적합 필드  | keyword   | text     |

## text 에 term 을 사용한다면?

---

**문제점**

- `text` 필드는 색인때 쪼개서 저장한다.
- 하지만 `term` 은 검색어를 쪼개지 않고 찾는다.
- 짝이 맞지않아 엉뚱한 결과가 나타난다.

예 : [개정판] 맞바람을

- `match` 로 subject(text) 검색 => 검색어도 개정판, 맞바람으로 쪼개져 => 저장 토큰과 맞음 => 검색됨
- `term` 로 subject(text) 검색 => 검색어가 [개정판] 맞바람 통짜를 찾음 => 그런 토큰이 없음 => 0건

## keyword 에 match 를 사용하면?

---

- keyword 필드엔 "안 쪼개는 analyzer" 가 들어있다.
- 그래서 `match` 를 사용해도 검색어를 쪼갤게 없어 통짜 그대로 쿼리한다. -> **term 과 결과가 같다.**
- 차이 : term 은 **점수(_score) 계산을 건너뛰고 "있다/없다" 만 본다.**
    - 그래서 관례상 완전일치/필터에는 term 을 쓴다.

## 정리

---

| 조합              | 결과                |
|-----------------|-------------------|
| keyword + term  | ✅ 정석 (완전일치)       |
| keyword + match | ✅ 잘 됨 (term 과 동일) |
| text + match    | ✅ 정석              |
| text + term     | ❌ 함정 (쪼갠 것 vs 통짜) |
