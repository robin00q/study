# 00. 참고자료 — 실제 인덱스 `title-group-meta-v2`

모든 레슨이 공통으로 참조하는 **교재용 실제 인덱스**다. WCS 작품그룹(작품 제목 묶음) 검색에 쓰인다. ES 7.5.
(헷갈릴 때 여기로 돌아와 대조한다.)

## 무엇을 검색하나
작품그룹 제목(`subject`)으로 검색한다. 예시 데이터(검색어 `맞바람` 기준):

| 분류 | 작품그룹 제목 예시 |
| --- | --- |
| 완전일치 (정규화하면 같음) | `맞바람`, `맞 바람`, `[개정판] 맞바람` |
| 전방일치 (입력으로 시작) | `맞바람 외전`, `맞바람 시즌2`, `맞바람: 그 후` |
| 포함 (중간/끝에 들어감) | `어느 날의 맞바람`, `비밀스러운 맞바람 이야기` |

## 매핑(mapping) — 필드가 어떤 타입인가

`subject` 하나를 여러 방식으로 동시에 색인한 **멀티필드** 구조:

```json
"subject": {
  "type": "text",                                  // 루트: 분석된 텍스트 (분석기 미지정 → standard)
  "fields": {
    "keyword": { "type": "keyword", "ignore_above": 256 },        // 원본 그대로 (완전일치/정렬)
    "korean":  { "type": "text", "analyzer": "korean" },          // 한국어 형태소(nori)
    "english": { "type": "text", "analyzer": "english" },         // 영문
    "ngram":   { "type": "text", "analyzer": "ngram_analyzer",
                 "search_analyzer": "standard" },                 // 부분일치 (색인≠검색 분석기!)
    "thai":    { "type": "text", "analyzer": "thai" }             // 태국어
  }
}
```

## 설정(settings) — 분석기가 실제로 어떻게 동작하나

```json
"analysis": {
  "tokenizer": {
    "korean_nori_tokenizer": { "type": "nori_tokenizer" },
    "ngram_tokenizer": { "type": "ngram", "min_gram": "2", "max_gram": "5" }
  },
  "analyzer": {
    "ngram_analyzer": { "tokenizer": "ngram_tokenizer", "filter": ["lowercase"] },
    "korean":  { "type": "custom", "tokenizer": "korean_nori_tokenizer", "filter": ["lowercase"] },
    "english": { "type": "custom", "tokenizer": "standard",
                 "filter": ["word_delimiter_graph","lowercase","english_stop","english_stemmer"] },
    "thai":    { "type": "custom", "tokenizer": "thai", "filter": ["lowercase","thai_stop"] }
  },
  "normalizer": {
    "lower_case_normalizer": { "type": "custom", "filter": ["lowercase","asciifolding"] }
  }
  // char_filter 없음! (괄호·공백을 제거하는 정규화가 색인 단엔 없다)
}
```

기타 index 설정: `number_of_shards: 1`, `number_of_replicas: 1`, `max_result_window: 30000`, `max_ngram_diff: 5`.

## 이 인덱스에서 꼭 기억할 4가지

1. **`subject`(루트)는 text**, 그 안에 **`subject.keyword`(원본 keyword)** 가 따로 있다.
2. **`ngram`은 색인은 `ngram_analyzer`(2~5글자 조각), 검색은 `standard`** — 색인≠검색 분석기.
3. **ngram tokenizer는 `min_gram:2, max_gram:5`** → 1글자/6글자+ 연속은 한계.
4. **char_filter가 없다** → 괄호·공백 "문자 제거" 정규화는 없음. 다만 검색 분석기(standard/nori)가 구두점 제거·공백 분할은 해줌.
