# 04. 분석기(Analyzer) 3단계

---

## 전체 플로우

---

```text
입력 문자열 -> [char_filter] -> [tokenizer] -> [token_filter] -> 최종 토큰들 -> 역색인 저장
              (문자 치환)        (자르기)         (토큰 다듬기)
```

## 단계별 역할

---

### char_filter (문자 필터) - 자르기 전, 원문의 글자를 치환/제거한다.

---

- 토큰화 이전 문자열 자체를 손본다.
- 예)
    - HTML 태그 제거 (\<b>맞바람\</b> => 맞바람)
    - 특정 문자 치환 (& => and)
- **0개 이상** 사용할 수 있다. (**없어도 된다.**)

### tokenizer (토크나이저) - 문자열을 토큰으로 자르는 핵심

---

- 여기서 "어떻게 쪼갤지" 가 결정된다.
- **정확히 1개**만 사용할 수 있다.
- 종류
    - `standard` : 공백 혹은 문장부호로 자른다.
        - [개정판] 맞바람 => 개정판, 맞바람
    - `nori_tokenizer` : 한국어 형태소로 자른다.
        - 안녕하세요 => 안녕, 하세요
    - `ngram` : N 글자씩 자른다.
        - 맞바람 => 맞바, 바람

### token_filter (토큰 필터) - 잘린 토큰들을 다듬기

---

- 토큰화 이후 각 토큰을 가공/추가/삭제
- 종류
    - lowercase (소문자화)
    - stop (불용어 제거 : the, a, ...)
    - synonym (동의어 추가)
- **0개 이상** 사용할 수 있다. (**없어도 된다.**)

## standard analyzer

---

standard analyzer 을 저번 섹션에서 아래와 같은 역할을 한다고 했다.

1. 공백/부호로 자르기
2. 소문자화

이걸 3단계로 매핑하면

- char_filter : 없음
- tokenizer : standard (공백/부호로 자름)
- token_filter : lowercase (소문자화)

## 역색인 - 색인된 토큰은 어디에 저장되는가?

---

**역색인 (inverted index)**

- 최종 토큰들이 저장되는곳이다.
    - 맞바람 -> [문서3, 문서9, 문서12]
    - 개정판 -> [문서3, 문서7]
- 검색 시 전체 문서를 훑지 않고, 이 표에서 토큰 줄만 보면 된다. -> 빠름
