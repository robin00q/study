# text vs keyword

---

## text 와 keyword 의 차이점

---

|         | text                             | keyword               |
|---------|----------------------------------|-----------------------|
| 분석(토큰화) | ✅ - `analyzer` 가 여러 토큰으로 쪼갠다.    | ❌ - 통째로 1개 토큰으로 저장된다. |
| 용도      | 전문검색(full-text) : 문장 속 단어로 찾기    | 정확일치, 필터, 정렬, 집계      |
| 정렬/집계   | ❌ - 쪼개져서 정렬 기준이 될 단일값이 없어 불가능하다. | ✅ - 가능하다.             |

## text 의 토큰화

---

**토큰화**

- text 가 어떻게 쪼개지는지는 `analyzer` 가 결정한다.
- text 라고 무조건 2글자씩 또는 형태소로 잘리는게 아니다.
    - 2글자씩 : 메로나 -> [메로, 로나]
    - 형태소 : 안녕하세요 -> [안녕, 하세요]

**standard(기본) analyzer 의 역할**

1. 공백 혹은 문장부호로 자른다.
2. 소문자화 한다.

## 실제 색인 예시

---

**[개정판] 맞바람** 을 `text`, `keyword` 로 색인하면 아래와같이 반영된다.

- text :
    1. `[개정판] 맞바람`에서 대괄호, 공백이 구분자로 제거된다.
    2. [`개정판`, `맞바람`] 딱 2개로 토큰화된다.
- keyword
    - 토큰나이저가 적용되지 않으므로, `[개정판] 맞바람` 그대로 색인된다.

## Keyword 의 가공

---

**normalizer**

- keyword 는 `tokenizer(쪼개기)` 는 없지만, `normalizer` 로 소문자화 혹은 공백정리는 가능하다.
- `analyzer`(text 용) 과 `normalizer`(keyword 용) 의 차이 = `tokenizer` 의 유무이다.

**ignore_above**

- keyword 는 `ignore_above` 가 넘는 긴 문자열은 색인을 스킵한다.
    - 256은 자동 생성되는 .keyword 서브필드의 기본값이며, 직접 명시한 keyword 는 기본이 **무제한**이다.
- 어떤 값은 keyword 에 안잡히지? 의 답이 될 수 있다.

## 멀티필드

---

- 한 필드가 text 와 keyword 를 모두 가질 수 있다.
- 같은 원본을 두 방식으로 동시에 색인하고, 상황에 맞게 골라쓰는것이다.
