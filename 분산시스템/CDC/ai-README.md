# CDC (Change Data Capture)

> 이기종 데이터 싱크의 핵심 메커니즘. "변경(change)"을 트랜잭션 로그에서 잡아 다른 시스템으로 전파한다.

## 공부 순서 & 진도
1~7을 순서대로. ⭐ 표시는 다음으로 넘어가기 전 반드시 자기 말로 설명할 수 있어야 하는 관문.

- [ ] 01 — 동기: 왜 CDC인가
- [ ] 02 — 변경을 잡는 3가지 방식
- [ ] 03 — 트랜잭션 로그 깊게 보기
- [ ] 04 — 도구와 아키텍처 (Debezium + Kafka)
- [ ] 05 — 초기 적재와 스냅샷
- [ ] 06 — 스키마와 데이터 타입
- [ ] 07 — 신뢰성과 운영

## 학습 방법
- 답을 찾기 전에 먼저 자기 말로 답해보기 (active recall)
- 막히는 질문은 번호로 Claude에게 질문 (예: "3-3 statement vs row 차이")
- 블록 다 채우면 Claude에게 퀴즈 요청
