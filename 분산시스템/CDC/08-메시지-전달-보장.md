# 메시지 전달 보장

---

## 전달 보장 : 왜 "정확히 한 번" 이 아니라 "최소 한 번" 인가?

---

Debezium/Kafka Connect 의 보장 = at-least-once (최소 한 번). 유실은 없지만 중복은 있을 수 있다.

**왜 exactly-once 가 안 되나?**

- Debezium 은 한 이벤트를 처리할 때 두 가지 쓰기를 한다.
    1. 이벤트를 Kafka 데이터 토픽에 발행
    2. 여기까지 했다 를 offset(my_connect_offsets) 에 커밋
- 이 둘은 원자적(atomic) 이지 않다.
    - **발행 성공** -> [여기서 죽으면] -> offset 커밋 못 함
    - **재시작** -> offset 은 옛날 위치 -> 이미 발행한 이벤트를 다시 발행 -> 중복!
- 순서를 뒤집으면(offset 먼저) 이번엔 유실이 생긴다. 유실 vs 중복 중 중복이 낫다(복구 가능하니까) -> at-least-once 를 선택한 것이다.

**중복인데 왜 문제가 안되나?**

- key = PK -> 다운스트림이 upsert 하면 같은 이벤트를 두 번 적용해도 결과 동일 = 멱등(idempotent)
- 이래서 "effectively-once(사실상 한 번)" 라고 부른다. 전달은 at-least-once + 적용은 멱등 -> 결과적으로 한 번

## 커넥터가 죽으면 - 유실? 중복?

---

**유실은 없고, 중복 가능성만 있다.**

- 유실이 없는 이유 :
    - offset 이 Kafka(my_connect_offsets) 에 영속 저장
    - binlog 는 DB 서버에 남아있음 -> **커넥터를 재시작**하면 offset 위치부터 다시 읽으면 됨.
- 중복이 생기는 경우 :
    - 발행은 했는데 offset 커밋 전에 죽은 틈에서 중복이 생길 수 있다.
    - 재시작 후 재처리하면 된다.

## binlog 가 읽기 전에 삭제되면 (retention 만료)

---

**대표적인 CDC 의 장애 시나리오이다.** 이러면 진짜로 유실된다.

- binlog 는 무한 보관이 아니라 expire (예 : binlog_expire_logs_seconds, 기본 30일) 로 삭제된다.
- 커넥터가 오래 멈춰 있었다면(장애, 유지보수) : 재시작 시 offset 이 가리키는 위치의 binlog 가 이미 삭제됨 -> 그 구간의 변경은 영원히 복구 불가
- 복구방법은 재스냅샷 뿐이다. 하지만 **중간 변경사항은 알 수 없다.**
    - offset 삭제 후 재스냅샷 (op=r)
    - 또는 signal 로 incremental snapshot 을 한다.

> 커넥터 중단 허용 시간 < binlog 보관 기간이 항상 성립해야 한다.

## 원본 DB 부하

---

**부하** : 생각보다 작다. log-based CDC 를 사용하기 때문

- Debezium 은 쿼리를 날리는게 아니라 replica 인 척 binlog 스트림을 받는다. -> 원본 부하는 replica 1대 추가 수준이다.
- polling 방식처럼 테이블을 반복 SELECT 하지 않으니 훨씬 가볍다.
- 예외적으로 부하가 큰 순간 :
    1. `initial snapshot (전체 읽기 + 글로벌락)`
    2. `incremental snapshot 의 chunk SELECT (조절 가능)`

## 모니터링 지표

---

| 지표           | 뜻                                               |
|--------------|-------------------------------------------------|
| CDC lag      | ts_ms - source.ts_ms (발생, 발행간 지연)               |
| binlog 잔여량   | 서버 최신 위치 - 커넥터 offset 위치 (retention 추월 예방)      |
| task 상태      | connector 말고 tasks[].state 까지 (RUNNING 을 믿지 마라) |
| consumer lag | 토픽 최신 offset - 컨슈머 커밋 offset                    |
