# 최신 CDC 방식

---

## log-based

---

개념

- db 의 트랜잭션 로그를 사용하여 CDC 를 수행한다.

**log-based 가 채택된 이유**

- 무부하 : 트랜잭션 로그를 읽기만 하므로 추가적인 부하가 없다.
- non-intrusive : DB 쓰기에 장애가 전파되거나 영향을 끼칠 요소가 없다. 추가 DB 스키마도 필요하지 않다.
- 적은 지연 - 준실시간으로 반영된다.

## 어디까지 읽었는지 어떻게 알까?

---

커넥터는 로그상의 위치를 저장한다. 이걸 영속화하므로 커넥터가 죽었다 살아나도 그 위치부터 읽는다.

- `MySQL` : binlog 파일명 + position 또는 GTID(전역 트랜잭션 ID)
- `PostgreSQL` : LSN(Log Sequence Number)

어디에 저장하는가?

- 분산모드 : 주로 Kafka 내부 토픽(connect-offsets) 에 저장한다.
- standalone : 주로 파일 혹은 Redis 에 저장한다.
