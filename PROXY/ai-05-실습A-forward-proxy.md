# 05 · 실습 A — forward 프록시 띄우기 (tinyproxy/squid + curl) — 질문

> 범위: 실제 forward 프록시를 도커로 띄우고 curl로 경유시켜 관찰. 답/결과·명령을 `05-...md`(내 노트)에 직접 기록.
> (명령·설정은 스스로 조사해 채우는 방식. 막히면 질문)

1. tinyproxy(또는 squid)를 도커로 띄우려면 어떤 이미지·포트·설정 파일이 필요한가? 조사해 기록.
2. `curl`에서 프록시를 지정하는 방법은? (`-x` 옵션, `HTTP_PROXY`/`HTTPS_PROXY` 환경변수 차이)
3. 프록시 경유로 IP 에코 서비스(`http://httpbin.org/ip` 등)를 호출하면 소스 IP가 무엇으로 보이나? **직접 호출 vs 프록시 경유**를 비교해 기록.
4. tinyproxy 로그에는 요청이 어떻게 남나? 한 줄 예시를 붙여넣고 필드 해석.
5. HTTPS URL을 프록시로 태울 때 `curl -v`로 보면 어떤 요청이 먼저 나가나? (`CONNECT` 관찰 — 03단계와 연결)
6. 허용/차단 규칙(Allow, Filter)을 설정해 특정 도메인을 막아보고, 차단 시 응답/로그를 기록.
7. (생각) 이 프록시는 클라이언트 설정이 필요한가? → forward의 특성과 연결(02단계).
